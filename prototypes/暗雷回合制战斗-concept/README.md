## 如何运行

### 步骤 1：打开 Godot 4.6

如果你还没有 Godot 4.6，从 https://godotengine.org/download 下载（免费、约 80MB）。

### 步骤 2：导入项目

1. 打开 Godot 4.6
2. 点击右上角 **"Import"** 按钮
3. 导航到 `C:\Users\suxiu\Desktop\my-game\prototypes\暗雷回合制战斗-concept\`
4. 选择这个文件夹
5. 点击 **"Import & Edit"**

> **没看到 import 按钮？** 直接 **Create New Project** → Project Path 指向这个文件夹 → 即可

### 步骤 3：创建主场景

1. 在 Godot 中按 **Ctrl+N**（或点击 Scene → New Scene）
2. 创建一个 **Node2D** 节点作为根
3. 在 FileSystem 面板中，点击根节点重命名为 `Main`
4. **附加脚本**：选中 `Main` 节点 → Inspector 面板 → Script 属性 → 点击文件夹图标 → 选择 `battle_prototype.gd`
5. **保存场景**：Ctrl+S，文件名为 `main.tscn`，保存到**当前文件夹**（`暗雷回合制战斗-concept/`）

### 步骤 4：设置主场景

1. Project → Project Settings → 左侧选 **"Application/Run"**
2. 找到 **Main Scene** → 选择你刚保存的 `main.tscn`
3. 点击 **Close**

### 步骤 5：运行

按 **F5**（或点击右上角 ▶ 按钮）

窗口会显示一个简单的战斗场景：

## 🧪 这个原型在测什么

### 核心假设（可证伪的）

> **如果玩家在回合制战斗中可以一键切换"手动模式"（逐回合决策）和"自动模式"（AI 接管），并且战斗中可以随时切换弹药类型（普通 / 电浆 / 跟踪），那么玩家会主动使用自动模式刷图，并主动切回手动模式打 Boss / 关键战斗。**

### 风险最高的假设

> **"手动 / 自动双模式切换"在 2D 像素回合制 RPG 中是真实可玩的，而不是"理论好听但实际割裂"。**

### 测的机制

- ✅ 1v1 回合制战斗
- ✅ 手动模式（点空格攻击，可选武器 / 弹药 / 防御）
- ✅ 自动模式（AI 每秒攻击一次）
- ✅ 战斗中**即时切换**手动 / 自动（不打断当前回合）
- ✅ 战斗中**即时切换**武器和弹药（回合间）
- ✅ 防御机制（减伤 50%）
- ✅ 3 武器 × 3 弹药 = 9 种组合
- ✅ 视觉反馈（HP 条、状态高亮、战斗日志）

### 不测的（被砍掉以节省时间）

- ❌ 暗雷遇敌 / 探索 / 移动
- ❌ NPC / 剧情 / 终端
- ❌ 图鉴 / 存档
- ❌ 多敌人 / 多场景
- ❌ 菜单 / 设置 / 标题
- ❌ 音效 / 音乐
- ❌ 教学
- ❌ 真实武器图鉴（占位符）
- ❌ 真实敌人图鉴（占位符）
- ❌ 机甲 4 部位系统
- ❌ 状态效果（燃烧 / 中毒等）

## 🆘 跑不通怎么办？

### 错误 1：场景无法打开
**症状**：按 F5 没反应 / 弹出错误
**原因**：可能没设置 Main Scene
**解决**：Project → Project Settings → Application/Run → Main Scene → 选 `main.tscn`

### 错误 2：黑屏
**症状**：窗口打开但全是黑的
**原因**：脚本没附加到节点
**解决**：选中 `Main` 节点 → Inspector → Script → 选 `battle_prototype.gd`

### 错误 3：控制台报错 "Identifier not declared"
**原因**：Godot 版本不对
**解决**：确认你用的是 **Godot 4.6**（不是 3.x 或 4.0-4.3）

## Findings（2026-06-12 第一次 Playtest）

### 假设验证结果
**PARTIALLY CONFIRMED**（部分确认）

- ✅ 手动 + 自动双模式能跑起来，模式切换流畅
- ✅ 1/2/3 选武器立即攻击比"先选武器 + 再按空格"更顺手
- ✅ 弹药切换功能可以工作
- ⚠️ 敌人 HP 200 / 武器伤害 20-50 → 战斗拖太长，玩家可能觉得"没奖励"
- ⚠️ 没试更多组合（导弹 + 跟踪弹、激光枪 + 电浆弹）

### 关键观察（玩家原话）
> "除了敌人有点难打，其他没有什么问题"

### 判定
**PROCEED**（继续）

机制本身可玩，进入"写 GDD 阶段"。具体平衡调优在 GDD 的 Tuning Knobs 段做，不在 prototype 硬调。

### 经验教训（写入概念文档的 GDD 注意事项）
- **平衡默认值**：敌人 HP 应该是 30-50（不是 200），让玩家 3-5 回合击杀小怪
- **操作简化**：1/2/3 选武器 = 攻击的反馈是"立即生效"，比"选 + 确认"两步更顺手
- **30 秒循环**：每个回合都有"我做了一件事"的具体反馈（看见伤害数字）
