# Game Concept: 钢轨猎人 / Railhunter

*Created: 2026-06-12*
*Status: Draft*

---

## Elevator Pitch

> **2300 年，银河系边缘一颗被遗弃的研究卫星变成走私者的天堂。** 你驾驶可定制机甲"漫游者"在卫星内部的废弃走廊中穿行，遭遇暗雷式敌人，搜刮武器与战利品，最终揭开卫星为何被遗弃的真相。
>
> 这是一款**回合制迷宫探索 RPG**，灵感来自 FC《重装机兵》的暗雷遇敌、武器收集与战车改造，搭配**现代独立游戏的高密度叙事节奏** —— 3-5 小时通关，每个房间都有回报，每个 NPC 都有故事，零废话。

| Aspect | Detail |
| ---- | ---- |
| **Genre** | Turn-based RPG / 迷宫探索 / 战车改造（机甲版） |

| **Platform** | PC（Steam / Epic / itch.io） |

## Core Fantasy

> **你是太空中一个孤独的考古学家/猎人** —— 在一座被时间遗忘的巨型轨道废墟中穿行，每一次推开未知的门、击败一个危险的敌人、修复一把古老武器，你都离"理解这里发生过什么"更近一步。
>
> **这游戏给你的，是其他游戏给不了的"发现的狂喜"**：你永远不会知道下一个转角后面是什么，但你会有强烈的动机去亲眼看看。当一个尘封 50 年的秘密在你的武器库填满、地图填满、剧情图谱填满的那一刻揭晓，那种满足感是不可替代的。
>
> **你不是在玩游戏，你是在"恢复一段失落的文明"** —— 这就是 Railhunter 的核心幻想。

---

## Unique Hook

> **像《重装机兵》**，**而且还有**：
> - **现代独立 RPG 的紧凑叙事节奏**（3-5 小时通关、零废话、每个 NPC 都有故事）
> - **武器 × 弹药的 build 深度**（同一把武器装不同弹药 = 完全不同打法）
> - **自动攻击节奏**（战斗永远不会"卡顿"，玩家永远有事在做）
> - **真相是收集的结果**（剧情不是被告知，是被拼出来的）

> **10 秒测试**："一款太空机甲回合制 RPG，每个房间都有回报，真相靠收集拼出来，3-5 小时通关。" —— 听过一次就懂，懂了就想去玩。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics (What the player FEELS)

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Discovery**（探索、秘密） | **1 (核心)** | 隐藏房间、机关、NPC 录音、终端日志、隐藏结局 |
| **Narrative**（戏剧、故事弧） | **2 (核心)** | 通过"收集"机制拼出真相，而非被动听 NPC 长对话 |
| **Fantasy**（扮演、身份） | **3 (支持)** | 你是机甲驾驶员"漫游者"，身份在游戏中逐渐揭示 |
| **Challenge**（障碍、精通） | **4 (支持)** | 武器 × 弹药组合的战术深度；boss 战需要正确 build |
| **Sensation**（感官愉悦） | **5 (支持)** | 像素艺术 + 太空氛围音效；战斗粒子效果；自动攻击的"心跳"节奏 |
| **Expression**（自我表达） | **6 (支持)** | 武器库 + 弹药组合 = 玩家自己的机甲 build |
| **Fellowship**（社交连接） | N/A | 单人游戏 |
| **Submission**（放松、舒适） | N/A | 这是高密度游戏，不主打放松 |

### Key Dynamics (Emergent player behaviors)

- 玩家会**反复试验武器 × 弹药组合**，直到找到"最对味"的打法
- 玩家会**主动回头搜刮已访问区域**（因为新武器能打开以前打不开的门）
- 玩家会**绘制自己的"剧情图谱"**（在脑内/外部工具中组织收集到的真相碎片）
- 玩家会**追求 100% 完成**（图鉴百分比是最强的回头率钩子）

### Core Mechanics (Systems we build)

1. **双模式回合制战斗（手动 + 自动）**
   - **手动模式**（默认 / 战术）：玩家逐回合决策，按菜单选择攻击 / 技能 / 道具 / 防御；适合有把握的战斗和 boss 战
   - **自动模式**（可一键切换 / 默认开）：AI 全程自动控制机甲行动（选最优攻击、用道具、防御），玩家观看；适合日常刷图、想减压、想刷级
   - **模式可在战斗中随时切换**（不打断当前回合）
2. **武器 × 弹药系统** —— 武器有插槽，可装不同类型弹药（普通 / 电浆 / 跟踪 / 爆破）；同一武器不同弹药 = 完全不同效果
3. **机甲升级系统** —— 机甲有 4 个部位（头 / 胸 / 臂 / 腿），每个部位独立 HP 和强化方向；自动模式下 AI 会优先保护受损部位
4. **暗雷迷宫探索** —— 移动时随机遇敌；地图密布隐藏房间、机关、宝箱
5. **收集驱动的叙事揭示** —— 通过 NPC 录音、终端日志、隐藏区域逐步拼出"卫星为何被遗弃"的真相
6. **图鉴系统** —— 武器图鉴、敌人图鉴、地区图鉴 —— 收集率显示在主菜单

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy**（自由、选择） | 武器 × 弹药 build 自由组合；探索路径自由选择；多种结局 | **Core** |
| **Competence**（精通、技能成长） | 战斗节奏的掌握；从"被虐"到"碾压"boss 的成长曲线 | **Core** |
| **Relatedness**（连接、归属） | 与"失落的卫星文明"产生情感连接（间接连接） | **Supporting** |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Explorers**（发现、理解系统、找秘密）— 核心玩家：地图 100% 探索、剧情拼图、隐藏结局
- [x] **Achievers**（目标完成、收集、进度）— 次要玩家：图鉴 100%、武器库全收集、boss 全挑战
- [ ] **Socializers**（关系、协作、社区）— 不适用（单人游戏）
- [ ] **Killers/Competitors**（主导、PvP、排行榜）— 不适用（无 PvP）

### Flow State Design

- **Onboarding curve（开篇 10 分钟）**：
  - 第 1 分钟：进入卫星，**遇到第一个"看起来"是普通房间但其实有隐藏要素的地方**（教会玩家"每个房间都值得看"）
  - 第 2-3 分钟：第一次暗雷遇敌，战斗界面教学（自动攻击、切换弹药、道具）
  - 第 5 分钟：第一次战利品掉落，给玩家一个"原来这就是回报"的明确信号
  - 第 10 分钟：第一次发现隐藏房间 / NPC 录音钩起第一个剧情问题
- **Difficulty scaling**：
  - 前 30 分钟：教学驱动，敌人弱、弹药少、build 单一
  - 中段：引入新弹药类型、新敌人种类（要换 build 才能打）
  - 后段：boss 战需要特定 build 组合（玩家主动实验）
- **Feedback clarity**：
  - **视觉反馈**：每次攻击有数字弹出（暴击 / 弱点 / 部位破坏都有不同颜色）
  - **图鉴反馈**：每次发现新东西都进图鉴，菜单里看到百分比上升
  - **剧情反馈**：每次收集到真相碎片，在"剧情图谱"中点亮一个节点
- **Recovery from failure**：
  - 战斗失败：返回上一个存档点（不丢武器、不丢剧情进度）
  - 迷路了：地图显示"已发现区域"，不显示完整地图（保留探索感）
  - 打不过 boss：可以回去刷弹药 / 换 build，没有"卡关"门槛

## Core Loop

### Moment-to-Moment (30 seconds)

**战斗中的一个回合**——根据玩家选择的模式，行为不同：

**手动模式**（战术向）：
- 玩家**逐回合决策**：攻击？用技能？用道具？防御？
- 弹药物品的切换是回合间进行（不消耗回合）
- **核心爽点**：build 试验的策略深度，boss 战的精准操作

**自动模式**（刷图 / 减压向）：
- 玩家**一键开启**，AI 全程自动控制机甲
- AI 按"最优策略"行动：选攻击最高的技能 / 选最有效的弹药 / 必要时防御或用道具
- 玩家**观看战斗动画**，可以随时切回手动模式接管
- **核心爽点**：**减压**——可以在休息时挂机刷图，刷经验 / 刷零件 / 刷金钱，不影响日常生活

**模式切换设计**：
- 战斗中按一个键**立即切换**（不打断当前回合）
- 战斗结束后默认回到**上次使用模式**
- 玩家可以设置"默认模式偏好"（设置里改）

### Short-Term (5-15 minutes)

**一个区域探索**：
- 进入新区域（"卫星表层 / 前哨基地走廊"）
- **目标**：找出这个区域的所有东西（隐藏房间、武器、宝箱、NPC 记录）
- **过程**：暗雷遇敌 → 战斗 → 胜利 → 获得新武器 / 弹药 → 装备 → 继续探索
- **完成信号**：地图上该区域"100% 探索"
- **"再来一把"钩子**：刚才打过的敌人里有一个变种 boss 还没发现；隔壁那个锁着的门需要新武器才能打开

### Session-Level (30-120 minutes)

**整个游戏分为 3 个"章节"**（每个章节 = 一个完整的卫星区域）：
- **章节 1：卫星表层（前哨基地）** —— 教学 + 第一种武器 + 第一个 NPC 故事（30-45 分钟）
- **章节 2：卫星中层（研究设施）** —— 第二种武器组合 + 中期剧情揭秘（45-60 分钟）
- **章节 3：卫星核心（反应堆 / 指挥中心）** —— 终极武器 + 真相揭晓（45-60 分钟）

**自然暂停点**：每个章节末尾的"章节结算屏"（显示本章节发现率、武器库、剧情进度）
**离场钩子**：每次离开游戏时，脑中会想"那个锁着的门我现在有了钥匙，明天一定要开"

### Long-Term Progression

**玩家成长方向**（按重要顺序）：
1. **武器库扩展** —— 收集更多武器和弹药组合（最多 20-30 种）
2. **机甲升级** —— 用战斗中获得的零件升级机甲的 4 个部位
3. **地图知识** —— 解锁区域越来越多，"已发现"百分比越来越高
4. **剧情拼图** —— 收集 NPC 录音 / 终端日志，逐渐拼出真相

**游戏"完成"信号**：找到**所有真相碎片** + 解锁**真结局**（取决于是否收集全图鉴）

### Retention Hooks

- **Curiosity（好奇）**：
  - 未探索的隐藏房间
  - 锁着的门 / 锁着的箱子（需要新武器 / 新弹药才能打开）
  - 谜题 / 机关（每次解开会揭示新通道）
  - 真相碎片散落在各处（玩家会想"还差几块就拼完了"）
- **Investment（投资）**：
  - 武器库、机甲升级、图鉴百分比都是永久进度
  - 剧情图谱是"你拼的"，放弃 = 前功尽弃
- **Mastery（精通）**：
  - 武器 × 弹药组合的 build 实验
  - boss 战的战术挑战
- **Completionism（100% 完成冲动）**：
  - 三个图鉴（武器 / 敌人 / 地区）+ 剧情图谱
  - 主菜单显示"X / Y 已发现"

## Game Pillars

### Pillar 1: 探索密度 (Exploration Density)

**定义**：每一个屏幕、每一个房间都必须有"被看见的回报" —— 一个新发现、一段新剧情、一件新武器、一个隐藏要素。

*设计测试*："我们在 X 房间没有放任何新东西，因为它只是连接两个区域的通道" → **错**。这违反支柱 1。给那个房间加点东西。

### Pillar 2: 发现 > 数值 (Discovery Over Numbers)

**定义**：玩家最满足的时刻来自"发现新东西"，而不是"看见数字变大"。

*设计测试*："这把武器的攻击力 +5%，但没有任何新发现" → **错**。要么给武器加新机制，要么去掉。

### Pillar 3: 每次战斗都是装备试验 (Every Fight Is A Build Test)

**定义**：武器 × 弹药组合让手动模式下的每次战斗都鼓励玩家尝试不同的 build；同时自动模式让不想 build 试验的玩家也能享受游戏。

*设计测试*："这个 boss 用普通弹就能过，玩家不需要换电浆弹" → **错**。调整 boss 让不同弹药组合有不同应对策略。

### Pillar 4: 真相是收集的结果 (Truth As Collection)

**定义**：核心剧情不是"被告知的"，而是"被拼出来的" —— 通过收集 NPC 录音、终端日志、隐藏房间逐步拼出真相。

*设计测试*："这段剧情用 NPC 对话告诉玩家" → **错**。应该让玩家自己探索发现。

### Anti-Pillars (What This Game Is NOT)

- **NOT 开放世界**：开放世界违背支柱 1。一个 100 平方公里的开放世界只能有 5% 的密度。我们要做的是**密度极高的紧凑世界**。
- **NOT 永久死亡 / Roguelike**：永久死亡违背支柱 4。玩家死了就要重新拼剧情，剧情进度会丢失。我们要的是**剧情和收集永久保存**。
- **NOT 多人 / 社交**：核心体验是"叙事与发现"——这是单人独享的体验。多加入就要砍剧情或加 UI 处理同步。
- **NOT 强制手动**：永远给玩家自动模式的选择权。手动模式是"我有讲究"，自动模式是"我想刷图 / 减压"，两者都尊重玩家的选择。刷怪升级是**有意义的玩家选择**，不是"无目的"。
- ~~NOT 无目的的刷怪升级~~（已修订）—— 自动模式让刷怪成为减压和升级的合理手段，不违背支柱 2。

## Technical Considerations

| Consideration | Assessment |
| ---- | ---- |
| **Recommended Engine** | **Godot 4**（推荐）—— GDScript 轻量、开源、回合制 RPG 友好、2D 像素完美支持、社区对独立游戏新人友好。第一次做开发者首选 |
| **Alternative** | Unity（C# 生态成熟，资源多，但学习曲线较陡）；Unreal（不推荐，对 2D RPG 来说太重） |
| **Key Technical Challenges** | (1) 暗雷遇敌的节奏控制（不能让玩家太烦也不能没有）；(2) 武器 × 弹药 build 的平衡（数据驱动设计）；(3) 地图"密度审计"工作流（每个房间都要过审） |
| **Art Style** | 像素艺术 + 太空科幻（参考《密特罗德》《极乐迪斯科》《Into the Breach》混合） |
| **Art Pipeline Complexity** | **Low-Medium** —— itch.io / OpenGameArt 上有大量免费商用像素资源（机甲、敌人、道具、地形）；必要时 AI 生成补足 |
| **Audio Needs** | **Moderate** —— 战斗节奏音乐 + 探索氛围音乐 + 关键剧情提示音；不追求动态自适应 |
| **Networking** | None（纯单机） |
| **Content Volume** | 3 个章节 / 30-50 个房间 / 20-30 种武器 / 10-15 种敌人 / 3-5 小时通关 |
| **Procedural Systems** | **最小化** —— 手工设计密度的核心，不要程序生成（程序生成会破坏密度） |

## Risks and Open Questions

### Design Risks

- **范围失控风险（高）** —— 第一次做游戏最容易踩的坑：总觉得"再多加一个房间"会变得更好
  - **缓解**：先把 MVP 做出来再加新内容
- **内容深度 vs 广度（高）** —— 密度是关键，少而精 > 多而稀
  - **缓解**：每个房间都过 2-3 次"密度审计"（这个房间有什么回报？）
- **谜题 / 机关设计（中）** —— 老 RPG 玩家的乐趣来源，但需要细心设计
  - **缓解**：用 3-4 种机关模板重复使用，加变化
- **剧情写作（中）** —— 第一次写要小心
  - **缓解**：用环境叙事为主（NPC 录音、终端日志、文件），对话少而精
- **战斗节奏"自动攻击"是否真的好玩（未验证）** —— 这是核心机制，必须先 prototype 验证
  - **缓解**：MVP 第一周先做战斗 prototype

### Technical Risks

- **范围 vs 时间的平衡** —— 几周内要 ship，必须严格控制 scope
- **引擎学习曲线** —— 第一次用 Godot 的话，第一周会有"什么都跑不起来"的挫败感
- **像素美术的视觉一致性** —— 自己画或 AI 生成都容易"风格漂移"

### Market Risks

- **饱和的 RPG 市场** —— RPG 是最饱和的品类之一
  - **缓解**：3-5 小时短篇 + 高密度的明确定位，避免和大作竞争
- **玩家期望值** —— FC《重装机兵》粉丝的期望可能很高
  - **缓解**：明确这是"致敬 + 现代化"，不是复刻

### Scope Risks

- **每个房间都要"有回报"的承诺** —— 这会让制作量翻倍
  - **缓解**：用"密度模板"（5-6 种回报模式重复使用）

### Open Questions

- **"自动模式 + 手动模式切换"是否真的好玩？** → 必须通过 prototype 验证。/prototype 是关键第一步
- **武器 × 弹药组合的数量边界？** → 经验值：5-8 种武器 × 3-4 种弹药 = 15-32 种组合，需要 prototype 测试"是不是 build 太多 / 太少"
- **隐藏结局的解锁条件**？ → 需要 prototype 完成后才能定具体条件

## MVP Definition

**Core hypothesis**: 玩家在**"自动攻击 + 弹药切换"的回合制战斗 + 高密度迷宫探索 + 收集驱动叙事**的组合下，能够持续 30-60 分钟不感到疲倦，并且会主动回头探索已访问区域。

**Required for MVP**：
1. **战斗核心 prototype** —— 手动模式 + 自动模式（可切换）+ 弹药切换 + 至少 3 种武器 / 3 种弹药
2. **1 个完整章节**（卫星表层）—— 8-12 个房间，每个房间有"被看见的回报"
3. **暗雷遇敌 + 战斗 + 战利品循环** —— 端到端跑通
4. **至少 1 个隐藏房间** —— 验证"密度 > 数量"的设计原则
5. **至少 1 段 NPC 录音 / 终端日志** —— 验证"真相是收集的结果"机制

**Explicitly NOT in MVP**（defer to later）：
- 多章节（先做 1 个）
- 完整剧情线（1-2 个真相碎片即可）
- 完整图鉴系统（先做武器图鉴）
- 机甲升级系统（先做 4 个部位 HP，不做升级）
- 隐藏结局（先做 1 个结局）

### Scope Tiers

| Tier | Content | Features | Timeline（solo） |
| ---- | ---- | ---- | ---- |
| **MVP** | 1 个卫星 / 1 个章节 | 战斗 prototype + 探索 + 收集 | **2-3 周** |
| **Vertical Slice** | 1 个完整章节 | MVP + 升级 + 1 段剧情 + 图鉴 | 1-2 个月 |
| **Alpha** | 全部 3 个章节，rough | 全部 features，rough | 2-3 个月 |
| **Full Vision** | 完整内容 | 全部 features，polished | 3-4 个月 |

## 视觉身份锚点 (Visual Identity Anchor)

> **"深空废墟中孤独的霓虹"** —— 在近乎完全黑暗的太空背景中，机甲、武器、敌人、NPC 都用**高饱和度的霓虹色**（电浆蓝 / 警告橙 / 紫红 / 毒绿）点亮，每一个发光元素都意味着"这里有故事"。

**视觉规则**：每一个发光元素必须有叙事 / 玩法意义。无意义的发光 = 视觉垃圾。

**支撑原则**：
- **黑暗 + 单一光源原则**：每个房间只用一个主光源（机甲的头灯 / 一扇破窗 / 应急灯），其他都是阴影
- **像素精度**：所有机甲和敌人用 32x32 像素基础单位，避免"画得不像像素"
- **冷暖对比**：太空背景冷蓝灰，前景角色暖橙红 —— 视觉上立刻分离

**色彩哲学**：背景是 90% 暗色 + 10% 高饱和点缀。这和"密度的承诺"一致 —— 在大面积的"未知"中，少数几个"已知"被强烈强调。
